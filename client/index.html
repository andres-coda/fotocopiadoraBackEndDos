BIENVENIDOS A LA API DE DELIBERYCOPY

Esta api existe para gestionar pedidos desde la app de delibery copy en todas sus versiones

Cuenta con distintas rutas de acceso a la información tanto para crear, editar, eliminar, o leer cualquiera de sus objetos

Rutas de acceso y formato de la información

libro

El metodo GET devuelve un arreglo de objetos con la siguiente extructura

    {
  • idLibro: number,
  • nombre: string,
  • descripcion: string,
  • edicion: string,
  • img: string,
  • cantidadPg: number,
  • autor: string,
  • deleted: boolean;
  • materia: { nombre: string, idMateria: number, deleted: boolean, },
  • stock:{ idStock:number, cantidad:number, deleted: boolean, estado: { estado: string, idEstadoPedido: number, deleted: boolean }, especificaciones: [ { nombre:string, idEspecificaciones: number, deleted:boolean } ], };
  • profeMaterias: { idProfeMateria:number, deleted:boolean, curso:{ idCurso:number anio:date, grado:string, deleted:boolean escuela: { idEscuela:number nombre:string, direccion:string, numero: string, email:string, deleted:boolean } } };
    • }

y otra lista de objetos dentro de "/users" con esta disposición:

    {
  • id: number;
  • name: string;
  • lastName: string;
  • user: string;
  • email: string;
  • password: string;
  • age: string;
    • }

Cada lista tiene metodos de lectura GET y GET+id, metodos de escritura POST, de edición PUT y de eliminación DELETE. También posee busqueda por Query, con cada uno de los campos en el caso de menu, y Query de user en la lista de users.

La api se carga en el puerto localhost:3000 y la base de dato, que es un elemento json se carga en el puerto localhost:3031 con la palabra clave db

La api esta tipada con tecnología "dto" va a ignorar cualquier dato que no corresponda con los objetos anteriormente mencionados, pero si falta algun campo mencionado, o esta tipado de diferente manera va a devolver un error 400. En la carga y la actualización las id de ambos objetos son ignoradas. Son generadas automaticamente por la api.
